Animationen

Mit CSS Elemente bewegen.

Ganz ohne JavaScript. Ohne ruckeln und zappeln. In jedem modernen Browser.

Animationen

  • @keyframes
  • animation-name
  • animation-duration
  • animation-timing-function
  • animation-iteration-count
  • animation-direction
  • animation-play-state
  • animation-delay
  • animation-fill-mode
  • animation (shorthand)

@keyframes


@keyframes falling {

    from {
      top: 0px;
    }

    to {
      top: 900px;
    }
}

Durch die CSS-at-Regel @keyframes kann der Autor die einzelnen Schritte einer CSS-Animationssequenz festlegen, indem er Keyframes (oder Wegpunkte) setzt, die während der Sequenz an bestimmten Punkten erreicht werden. Das ermöglicht eine spezifischere Kontrolle über die Zwischenschritte einer Animationssequenz gegenüber dem Ergebnis einem dem Browser überlassenen, automatisch berechneten Ablauf.

@keyframes


@keyframes keyframes1 {

    0% {
      top: 0px;
    }

    100% {
      top: 700px;
    }
}

Statt der Schlüsselworte "from" und "to" kann man auch die Prozentwerte der Strecke angeben.

@keyframes


@keyframes keyframes2 {
    0% {
	top: 0px;
    }
    10% {
	top: 400px;
    }
    90% {
	top: 500px;
    }
    100% {
	top: 900px;
    }
}

Statt der Schlüsselworte "from" und "to" kann man auch die Prozentwerte der Strecke angeben.

@keyframes


@keyframes keyframes3 {
    0% {
	top: 0px;
	opacity: 0.2;
	
    }
    50% {
	top: 400px;
	opacity: 1;
    }
    100% {
	top: 900px;
	opacity: 0.2;
    }	
}

Neben der POsition können wir auch andere Attribute ändern. Beispielsweise die Opacity.

@keyframes


@keyframes keyframes4 {
    0% {
	top: 0px;
	opacity: 0.2;
	transform: translateX(-80px);    
    }
    25% {
	transform: translateX(40px);
    }
    50% {
	opacity: 1;
	transform: translateX(-80px);    
    }
    75% {
	transform: translateX(40px);
    }
    100% {
	top: 1000px;
	opacity: 0.2;
	transform: translateX(-80px);
    }	
}

Und wir können auch die Position in einer anderen Dimension verschieben.

@keyframes


@keyframes keyframes5 {
0% {
	transform: translateX(-80px) translateY(0px);    
    }
    50% {
	transform: translateX(40px) translateY(500px);    
    }
    100% {
	transform: translateX(-80px) translateY(1000px);
    }		
}

    }
    100% {

    }		
}

Selectoren

Selektoren definieren, auf welche Elemente eine Reihe von CSS Regeln angewendet wird.

:nth-child(an+b)


	element:nth-child(an + b) { style properties }
  • 1n+0, oder einfach n, trifft auf jedes Kindelement zu.
  • 2n+0, oder einfach 2n, würde auf die Kindelemente 2, 4, 6, 8, etc. zutreffen. Hier kann auch einfach das Schlüsselwort even verwendet werden.
  • 2n+1 würde auf die Elemente 1, 3, 5, 7, etc. zutreffen. Hier gibt es auch eine Kurzform: Das Schlüsselwort odd.
  • 3n+4 würde auf die folgenden Elemente zutreffen: 4, 7, 10, 13, etc.

Weiterlesen: MDN: CSS Selectors

:nth-child() erstmal einfach


i {
    animation-duration: 5s;
}
i:nth-child(2) {
    animation-duration: 10s;
}
i:nth-child(3) {
    animation-duration: 15s;
}

:nth-child() dann mal richtig


i:nth-of-type(5n)    {height:30px; width:30px;}
i:nth-of-type(5n+1)  {height:24px; width:24px;}
i:nth-of-type(5n+2)  {height:10px; width:10px;}...

i:nth-of-type(3n)    {animation-delay: 2.3s;}
i:nth-of-type(3n+1)  {animation-delay: 1.5s;}
i:nth-of-type(3n+2)  {animation-delay: 3.4s;}

i:nth-of-type(7n)    {opacity: 0.5;}
i:nth-of-type(7n+1)  {opacity: 0.8;}
i:nth-of-type(7n+2)  {opacity: 0.3;} ...

i:nth-of-type(11n) {animation-timing-function: ease-in-out;}
i:nth-of-type(11n+1) {animation-timing-function:ease-out;}
i:nth-of-type(11n+2) {animation-timing-function:ease;} ...
i:nth-of-type(11n+5) {animation-timing-function:
                         cubic-bezier(0.2, 0.3, 0.8, 0.9);}
